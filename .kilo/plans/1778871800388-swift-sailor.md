# Plan: Make HiBuddy Fully Functional

## Date: 2026-05-16

## Current State Assessment

The backend is **~70% complete** (all API endpoints exist, models, services, auth, task scheduler are implemented). The Kotlin frontend is a **UI prototype only** — all screens use hardcoded `SampleData` instead of calling the API. The ViewModels and ApiService/repositories DO exist and ARE wired to the backend, but the Compose screens ignore them entirely.

---

## CRITICAL GAPS — Frontend (Kotlin/Android)

### 1. No Chat Screen at All (Highest Priority)
- **Problem**: Tapping a match row does nothing. No chat detail screen exists. No WebSocket client in the app.
- **Fix**: Create `ChatScreen.kt` with message list + input bar. Create `ChatViewModel.kt` that loads messages via REST and connects via WebSocket. Wire `MatchesScreen` clicks to navigate to `ChatScreen`.

### 2. DiscoverScreen Uses Hardcoded SampleData
- **Problem**: `DiscoverScreen.kt` uses `SampleData.users`/`SampleData.projects` instead of `DiscoverViewModel.uiState`. Swipes manipulate local lists, never call the API.
- **Fix**: Rewrite `DiscoverScreen` to use `DiscoverViewModel`. Replace local card stacks with `uiState.userCards`/`uiState.projectCards`. Call `viewModel.swipe(action)` on swipe gestures. Remove `SampleData` imports.

### 3. MatchesScreen Uses Hardcoded SampleData
- **Problem**: `MatchesScreen.kt` uses `SampleData.matches` instead of `MatchesViewModel.uiState`. Click handlers are empty comments `/* Open Chat */`.
- **Fix**: Rewrite `MatchesScreen` to use `MatchesViewModel`. Wire click handlers to navigate to `ChatScreen`. Remove SampleData imports.

### 4. TasksScreen Uses Hardcoded SampleData
- **Problem**: `TasksScreen.kt` uses `SampleData.tasks`/`SampleData.myProjects`. The "+" button does nothing. Tasks can't be created, moved between columns, or edited.
- **Fix**: Rewrite `TasksScreen` to use `TasksViewModel`. Wire column drag/drop or tap to change task status via `updateTaskStatus()`. Wire "+" button to a create-task dialog or screen.

### 5. No Create Project Screen
- **Problem**: Users can only swipe on existing projects. No way to create their own projects from the app.
- **Fix**: Create `CreateProjectScreen.kt` with form fields matching `CreateProjectRequest`. Call `ProjectRepository.createProject()`.

### 6. No Project Detail Screen
- **Problem**: Can't view project details, members, dashboard stats, or manage members.
- **Fix**: Create `ProjectDetailScreen.kt` showing project info, member list, dashboard stats, and task list for that project.

### 7. No Navigation for Deep Screens
- **Problem**: `MainActivity.kt` has 4 bottom tabs but no navigation stack for chat, project detail, create project, create task.
- **Fix**: Replace simple `Screen` enum with a proper navigation solution (NavHost/NavController) that supports push/pop to chat screens, project details, etc.

## BACKEND BUGS & GAPS

### 8. auth_service.py: resend_verification Has Broken Code
- **File**: `backend/app/services/auth_service.py:192-194`
- **Problem**: `select(func.count()).select_from(__import__('sqlalchemy.sql.functions', fromlist=['func']))` is garbage code
- **Fix**: Remove or replace with proper day-limit check on resend (track last resend time on User model)

### 9. profile.py: UserProfile relationship missing `back_populates` on many models
- **Problem**: `UserProfile` has `relationship("User", back_populates="profile")` but `UserCompletedCourse.user` relationship is missing from the model

### 10. No file/image upload endpoint
- **Problem**: MinIO is configured but unused. Users can't upload avatars or project thumbnails.
- **Fix**: Add `POST /api/v1/upload/avatar` and `POST /api/v1/upload/thumbnail` endpoints backed by MinIO.

### 11. Email sending is stubbed
- **Problem**: `send_verification_email` and `send_reset_code_email` are empty `pass` functions.
- **Fix**: Implement SMTP email sending or integrate an email service (SendGrid, Resend, etc.)

### 12. Discover uses Jaccard similarity, not Milvus embeddings
- **Problem**: `_calculate_text_similarity` uses word overlap, ignoring the Milvus vector index that was set up for this purpose.
- **Fix**: Use `_search_similar_projects()` / `_search_similar_users()` that query Milvus collections with the user's embedding vector.

### 13. No user search endpoint
- **Problem**: Can't search for users by name, skill, or role.
- **Fix**: Add `GET /api/v1/search/users?q=&skill=&role=` endpoint.
